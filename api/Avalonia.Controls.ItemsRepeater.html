Events

ElementClearing

Occurs each time an element is cleared and made available to be re-used.

public event EventHandler<ItemsRepeaterElementClearingEventArgs>? ElementClearing

Event Type

EventHandler<ItemsRepeaterElementClearingEventArgs>

Remarks

This event is raised immediately each time an element is cleared, such as when it falls outside the range of realized items. Elements are cleared when they become available for re-use.

ElementIndexChanged

Occurs for each realized Avalonia.Controls.Control when the index for the item it represents has changed.

public event EventHandler<ItemsRepeaterElementIndexChangedEventArgs>? ElementIndexChanged

Event Type

EventHandler<ItemsRepeaterElementIndexChangedEventArgs>

Remarks

When you use ItemsRepeater to build a more complex control that supports specific interactions on the child elements (such as selection or click), it is useful to be able to keep an up-to-date identifier for the backing data item.

This event is raised for each realized Control where the index for the item it represents has changed. For example, when another item is added or removed in the data source, the index for items that come after in the ordering will be impacted.
